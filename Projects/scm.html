ShipShape (Ongoing)

A collaboration between professors in Gaana and Arizona State University resulted in an idea to create an experiental learning mechanism that accelerates supply chain knowledge for practioners in developing countries.

Design Process

Studying and Testing TableTop Games
ASU’s Supply Chain Professor created these inital paper prototypes of the games through which
  • His idea and concepts of the game were communicated to the designers
  • The paper prototypes were used for initial user testing qualitiative factors like -
    • How much instructions/information to give beforehand
    • Which situations/rewards make sense and how much the users care about them
Examples of other useful Insights -
“So if I mess this up, patients are goint to die? That doesn’t seem real”
“Usually wouldn’t I be able to order even if another order is on the way?”
“It just feels like i’m guessing my way through this”
Designing Game Mechanics
Keeping in mind the feedback from the testing of the paper prototypes, digital versions of those games were added. The game has it’s own storyline and characters to keep the user’s engaged. The design team ideates on multiple ideas, building on top of eachother until there is one or two versions that are tested on by users.
Testing Mockups
The mockups were tested mostly by students including Ghanian students which was the location for the first Beta test. User testing sessions were conducted with the goal of each session decided beforehand. Some of the things we were able to figure out throught these sessions were:
  • Ensuring the language of the instructions were simple and correctly understood
    •  
  • Ensuring that level difficulty is set correctly
    •  
  • Fun factor of the game - making note of boring aspects of the game
    •  
  • Meausuring how much the players learnt from the game

Feedback from these sessions is recorded and grouped into actionable items if a pattern is found

Learnings

  • Converting extremely technical concepts into a simple teachable games.
  • Game Design techniques, along with building a narrative, developing characters, creating a model for increasing level difficulty, and reward systems.
  • Working with a restrained screen size and memory capacity.
